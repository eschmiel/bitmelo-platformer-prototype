Animations

Bit Rhythm allows you to take a set of sprites from the tile sheet and create an animation out of them. You can then feed these animations into an animation controller that will run the appropriate animation for an entity based on the entity’s state.

Animation Objects

In order to create an animation, we use the Animation class to create an animation object and pass it the sprites and timings the animation will consist of as parameters. We then use this object to run the animation whenever we need it.

`var animationName = new Animation([[tileID, timing],

                                           [tileID, timing]]);`

THIS IS IMPORTANT: You must pass a two dimensional array(an array consisting of arrays) to the new Animation object when you create it. Each position in the first array is a single animation frame of the animation. The second dimension of the array holds two values. The first value is the tile id of the sprite for that animation frame. The second value is how many update frames the animation should sit on that animation frame for before continuing to the next frame.

Note: Bitmelo averages 60 update frames per second, so if you would like an animation frame to sit for 1/10th of a second before moving to the next frame of the animation, you should set the second value to 6.

EXAMPLE: This code creates a simple walk animation out of 4 animation frames that lasts for 1/3rd of a second.

`var walk = new Animation([[227, 5], [228, 5], [225, 5], [226, 5]]);`

Animation Methods

Animation objects have two methods.

runAnimation() - This method returns the current sprite of the animation.

Each time runAnimation() is called, it iterates the animation through an update frame of the animation and will automatically switch animation frames according to the set timing.

The AnimationController will make use of this method to animate the entity it manages and you should typically let this handle running the animation.

resetAnimation() - This method resets the animation so that it can be run from the beginning.

Animation Controllers

The Animation Controller manages the animations of an Entity and switches between them based on the Entity’s state. Every Entity is created with an Animation Controller.

You will typically interact with the Animation Controller through its 3 main methods: add(state, animation), setState(state), and animate(posX, posY, flip). The typical workflow for working with Animation Controllers is as follows:

  1. In the constructor of an entity, you will add animations to the Animation Controller by calling add on the entity’s Animation Controller and passing two parameters to it. The first parameter is the state associated with the animation (this should be a string like “idle” or “walk”.) The second parameter is the animation object you would like to add to the Animation Controller. You may wish to create the animation object itself when calling add.
  2. When appropriate according to the logic of the program, you will set the current state of the Animation Controller by calling setState on it, passing its new state as a parameter.
  3. In the game’s core update loop, you will call animate on the Animation Controller of the Entity you wish to draw and pass to it the x position and y position of the Entity as parameters. You may also pass whether or not to flip the sprite of the animation as a third parameter. This will draw the animation for the Animation Controller’s current state on the screen.

Animation Controller Methods

Animation Controllers have 4 methods.

add(state, animation) - This method adds an animation object and its assigned state to the Animation Controller. When the Animation Controller’s state is set to the one tied to the added animation object, the Animation Controller will draw that animation when animate() is called on it.

setState(state) - This method sets the current state of the Animation Controller. The Animation Controller will draw the animation tied to the currently set state when animate() is called on the Animation Controller.
